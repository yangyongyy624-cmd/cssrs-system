# C-SSRS 电子评估系统

> Columbia Suicide Severity Rating Scale — 基于 FastAPI 的电子化自杀风险评估工具

一个完整的自杀风险评估电子化解决方案，支持患者扫码答题、自动评分、风险分层，以及云端-本地分离架构保障数据隐私。

## 项目愿景

**让自杀的人越来越少。**

C-SSRS 是国际公认最广泛使用的自杀风险评估工具，但传统的纸质量表存在评分慢、易遗漏、数据统计困难等问题。本项目将其电子化，目标是：

- **降低基层使用门槛**：任何社区工作者、乡村医生，只需一个二维码就能开展标准化评估
- **提高响应速度**：自动评分 + 风险分层，无需专业训练即可识别高危个案
- **数据驱动科研**：结构化存储评估数据，支持流行病学研究和干预效果评估
- **保护患者隐私**：数据仅存本地，云端零感知，从架构上杜绝泄露

适用于：精神科门诊、社区心理服务、学校心理咨询、危机干预热线。

## 特点

- **隐私优先**：所有评估数据存储在本地，云端仅做转发
- **自动评分**：基于 C-SSRS 标准的 5 层评分系统
- **风险分层**：无/低/中/高/极高 5 级风险自动判定
- **多端访问**：医生端网页 + 患者端 H5 + 语音管理
- **离线可用**：患者端 PWA 支持添加到手机桌面
- **零成本部署**：无需服务器，一台旧 Mac / 树莓派即可运行

## 架构

```
患者手机 → 云端轻量网关 → SSH 隧道 → 本地 Mac mini (数据仓库)
          (仅转发)                  (数据存储 + 评分)
```

### 组件说明

| 组件 | 位置 | 功能 |
|------|------|------|
| 云端网关 | Ubuntu 服务器 | 认证、转发、二维码生成 |
| 本地服务 | Mac mini | 数据存储、评分计算、报告生成 |
| SSH 隧道 | 自动建立 | 加密传输评估数据 |

## 快速开始

### 环境要求

- Python 3.10+
- Ubuntu 22.04+ (云端)
- macOS (本地)

### 安装

```bash
# 克隆仓库
git clone https://github.com/yangyongyy624-cmd/cssrs-system.git
cd cssrs-system

# 安装依赖
python3 -m venv .venv
source .venv/bin/activate
pip install -r backend/requirements.txt
```

### 本地运行

```bash
# 启动服务
bash manage.sh start

# 查看状态
bash manage.sh status

# 停止服务
bash manage.sh stop
```

服务启动后访问：
- 医生端：http://localhost:8000/
- 患者端：http://localhost:8000/patient.html

### 云端部署

1. 配置服务器（参考 `docs/deployment.md`）
2. 设置防火墙（开放端口 22, 8888）
3. 部署代码并启动服务
4. 建立 SSH 反向隧道

详细步骤见 `docs/` 目录。

## 评分系统

### 5 层评分模型

| 层级 | 维度 | 范围 | 说明 |
|------|------|------|------|
| 筛查 | 自杀意念 | 阴性/阳性 | C-SSRS 筛查问题 |
| 意念严重度 | 最高级别 | 0-5 | 从渴望死亡到有计划有意图 |
| 意念强度 | 5 维度总分 | 0-25 | 频率、持续时间、可控性、阻止因素、现实性 |
| 行为 | 自杀行为 | 5 类 | 实际企图、中断、中止、准备行为、非自杀性自伤 |
| 致死性 | 企图致死性 | 0-6 | 基于医疗干预程度 |

### 风险分层

| 等级 | 标签 | 条件 |
|------|------|------|
| none | 无风险 | 筛查阴性 |
| low | 低风险 | 仅渴望死亡，无计划意图 |
| medium | 中风险 | 有方法的主动意念 |
| high | 高风险 | 有计划或有意图 |
| critical | 极高风险 | 有计划且有意图 |

## 使用方式

### 医生端

1. 管理员生成医生访问码（4 位 PIN）
2. 医生手机浏览器扫码认证
3. 新建评估，生成患者二维码
4. 查看评分结果和患者历史

### 患者端

1. 微信扫码或浏览器扫码
2. 输入 6 位访问码
3. 逐题作答（约 2-3 分钟）
4. 提交完成

### 管理员

通过语音助手管理：
- 生成评估码
- 管理医生权限
- 查看评估摘要

## 数据安全

- 所有评估数据仅存储在本地
- 云端不保留任何评估内容
- 数据传输通过 SSH 加密隧道
- 医生端 PIN 认证，支持随时撤销
