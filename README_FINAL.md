# C-SSRS 电子评估系统 — 最终版

> Columbia Suicide Severity Rating Scale — 基于 FastAPI 的电子化自杀风险评估工具
>
> **版本**：2026.06.04 最终版
> **作者**：杨勇（Yang Yong），武汉市普仁医院精神科

---

## 项目愿景

**让自杀的人越来越少。**

C-SSRS 是国际公认最广泛使用的自杀风险评估工具，但传统的纸质量表存在评分慢、易遗漏、数据统计困难等问题。本项目将其电子化，目标是：

- **降低基层使用门槛**：任何社区工作者、乡村医生，只需一个二维码就能开展标准化评估
- **提高响应速度**：自动评分 + 风险分层，无需专业训练即可识别高危个案
- **数据驱动科研**：结构化存储评估数据，支持流行病学研究和干预效果评估
- **保护患者隐私**：数据仅存本地，云端零感知，从架构上杜绝泄露

适用于：精神科门诊、社区心理服务、学校心理咨询、危机干预热线。

## 系统架构

```
患者手机 → 云端网关 (82.156.238.242:8888) → SSH隧道 (8889) → Mac Mini (8000)
医生手机 → 云端网关 (82.156.238.242:8888) → SSH隧道 (8889) → Mac Mini (8000)
```

| 组件 | 位置 | 功能 |
|------|------|------|
| 云端网关 | Ubuntu 服务器 | Nginx 纯转发，几 MB 内存 |
| SSH 隧道 | 自动建立 | 加密传输评估数据 |
| 本地服务 | Mac Mini | FastAPI + SQLite，数据存储 + 评分 |

---

## 访问入口

| 入口 | 认证 | 功能 |
|------|------|------|
| `/mobile` | 4 位医生 PIN | 医生端：新建评估、查看列表、查看报告 |
| `/admin` | 4 位管理密码 (8888) | 管理后台：创建/撤销医生 PIN |
| `/` | → 自动重定向到 /mobile | 强制 PIN 认证 |
| `/patient.html` | 无需认证 | 患者扫码答题 |

## 快速开始

### 医生端
1. 手机扫「C-SSRS医生入口二维码」
2. 输入 4 位 PIN（首次）
3. 点击 ＋ 新建评估
4. 患者扫码答题
5. 自动查看报告

### 管理后台
1. 手机扫「C-SSRS管理后台二维码」
2. 输入管理密码 8888
3. 创建/撤销医生 PIN

## 评分系统

### 5 层评分模型

| 层级 | 维度 | 范围 |
|------|------|------|
| 筛查 | 自杀意念 | 阴性/阳性 |
| 意念严重度 | 最高级别 | 0-5 |
| 意念强度 | 5 维度总分 | 0-25 |
| 行为 | 自杀行为 | B1-B5 |
| 致死性 | 企图致死性 | 0-6 |

### 风险分层

| 等级 | 条件 |
|------|------|
| 无风险 | 筛查阴性 |
| 低风险 | 仅渴望死亡 |
| 中风险 | 有方法的主动意念 |
| 高风险 | 有计划或有意图 |
| 极高风险 | 有计划且有意图 |

## 数据安全

- 所有评估数据仅存储在本地 Mac Mini
- 云端 Nginx 仅做转发，不保存任何患者数据
- 数据传输通过 SSH 加密隧道
- 医生端 PIN 认证，支持随时撤销
- GitHub 仓库已脱敏（无 IP、PIN、密码）
